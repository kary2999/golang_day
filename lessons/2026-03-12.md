# Day 1 — 变量声明与基本类型

> 日期：2026-03-12

## 📌 今日知识点

Go 是**强类型语言**，变量类型在编译期确定，不像 PHP 那样随意赋值。

---

## PHP vs Go 对比

```php
// PHP - 弱类型，随便写
$name   = "Alice";
$age    = 25;
$price  = 9.99;
$active = true;
```

```go
// Go - 方式一：显式声明
var name   string  = "Alice"
var age    int     = 25
var price  float64 = 9.99
var active bool    = true

// Go - 方式二：短变量声明（推荐，编译器自动推断类型）
name   := "Alice"
age    := 25
price  := 9.99
active := true
```

---

## 基本类型速查

| Go 类型 | PHP 类比 | 说明 |
|---------|---------|------|
| `string` | `string` | 字符串 |
| `int` / `int64` | `int` | 整数 |
| `float64` | `float` | 浮点数 |
| `bool` | `bool` | 布尔值 |

---

## ⚠️ 最容易踩的坑：类型不自动转换

```go
age   := 25
price := 9.99

// ❌ 直接运算会报错
total := price * age  // 编译错误：mismatched types

// ✅ 需要显式转换
total := price * float64(age)
```

PHP 会自动帮你转，Go 不会，必须手动。

---

## 💡 高精度计算（做市/加密货币必看）

Go 的 `float64` 和 PHP 一样有 IEEE 754 精度问题：

```go
fmt.Println(0.1 + 0.2)        // 0.30000000000000004
fmt.Println(0.1 + 0.2 == 0.3) // false ！
```

解决方案：使用 `shopspring/decimal`（类比 PHP 的 bcmath）

```go
import "github.com/shopspring/decimal"

a, _ := decimal.NewFromString("0.1")
b, _ := decimal.NewFromString("0.2")
fmt.Println(a.Add(b))                          // 0.3 ✅
fmt.Println(a.Add(b).Equal(decimal.NewFromString("0.3"))) // true ✅
```

---

## 📝 今日小结

Go 变量类型严格、不自动转换，`:=` 是最常用的声明方式；涉及金融精度，永远别用原生 float。
