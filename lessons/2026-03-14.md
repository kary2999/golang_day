# Day 5 · 接口（Interface）

> 日期：2026-03-14

## 🧠 核心概念

Go 的接口是**隐式实现**的——只要一个类型拥有接口要求的所有方法，它就自动满足该接口，无需声明 `implements`。

这与 PHP 的显式接口完全不同：

### PHP 写法
```php
interface Shape {
    public function area(): float;
}

class Circle implements Shape {
    public function __construct(private float $r) {}
    public function area(): float {
        return M_PI * $this->r ** 2;
    }
}
```

### Go 写法
```go
type Shape interface {
    Area() float64
}

type Circle struct{ R float64 }

// 不用声明 implements！只要有 Area() 方法就满足接口
func (c Circle) Area() float64 {
    return math.Pi * c.R * c.R
}
```

## 🛠 实用例子

```go
type Logger interface {
    Log(msg string)
}

type FileLogger struct{}
func (f FileLogger) Log(msg string) { fmt.Println("[FILE]", msg) }

type StdoutLogger struct{}
func (s StdoutLogger) Log(msg string) { fmt.Println("[STDOUT]", msg) }

func Process(l Logger, data string) {
    l.Log("处理: " + data)
}

// 随意替换实现，Process 无感知
Process(FileLogger{}, "订单#123")
Process(StdoutLogger{}, "订单#456")
```

Go 的接口让**依赖注入**和**单元测试**变得非常简洁——传什么实现都行，只要"形状"对。

## 📌 今日小结

> Go 接口是"鸭子类型 + 编译期检查"，比 PHP 更灵活，比动态语言更安全。
