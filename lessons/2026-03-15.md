# Day 6 · Goroutine — 轻量级并发

> 日期：2026-03-15

## 知识点：Goroutine 基础

Go 的并发核心是 **goroutine**——比线程轻得多的协程，用 `go` 关键字启动，成本极低（初始栈约 2KB）。

---

## PHP 对比

```php
// PHP：没有原生协程，通常靠多进程/Swoole
// Swoole 示例（需扩展）
go(function () {
    echo "异步任务\n";
});
```

```go
// Go：内置，零依赖
go func() {
    fmt.Println("异步任务")
}()
```

Go 的 `go` 是语言关键字，不是库，任何函数前加 `go` 即可并发执行。

---

## 实用例子：并发抓取多个 URL 耗时统计

```go
package main

import (
    "fmt"
    "sync"
    "time"
)

func fetch(url string, wg *sync.WaitGroup) {
    defer wg.Done()
    time.Sleep(100 * time.Millisecond) // 模拟网络
    fmt.Println("done:", url)
}

func main() {
    var wg sync.WaitGroup
    urls := []string{"api/a", "api/b", "api/c"}

    start := time.Now()
    for _, u := range urls {
        wg.Add(1)
        go fetch(u, &wg)
    }
    wg.Wait() // 等所有 goroutine 完成
    fmt.Printf("总耗时 %v（串行需 300ms）\n", time.Since(start))
}
// 输出：总耗时 ~100ms
```

**关键点：**
- `sync.WaitGroup` 用于等待所有 goroutine 完成
- `defer wg.Done()` 确保无论如何都会通知完成
- 3 个任务并发跑，总耗时约等于最慢的那个
